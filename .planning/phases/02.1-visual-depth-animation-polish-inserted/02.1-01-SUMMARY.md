# Phase 2.1 Plan 01: Visual Depth (VIS-DEPTH) Summary

**Clamped-minimum-gap shell radius falloff (replaces `Math.pow(0.85, idx)`) plus a per-shell `weight` legibility scalar on the Scene contract, rendered through paint so old shells fade with age while every ring stays individually distinguishable to the core.**

## Performance

- **Duration:** ~10 min (continuation/finalize run only; Tasks 1–2 in prior session)
- **Started:** 2026-06-19 (Tasks 1–2), finalized 2026-06-20
- **Completed:** 2026-06-20
- **Tasks:** 3 (2 auto/TDD + 1 human-verify checkpoint)
- **Files modified:** 4

## Accomplishments
- **D-01 (no central blob):** Replaced the inline `radius = Math.pow(0.85, idx)` with a pure `shellRadius(idx, N, prevRadius)` clamped-minimum-gap hybrid — an exponential term (`RADIUS_EXP_BASE=0.92`) blended 0.45/0.55 with a linear even-spacing term, then hard-floored so each ring sits ≥ `MIN_GAP` (`0.9/N`) inside its predecessor. Frontier (idx 0) is largest; every adjacent pair is guaranteed separated.
- **D-02 (fade with age, keep highlights):** Added a pure `shellWeight(idx, N, conflict, energyProxy)` — age fade (older = dimmer, `lerp(1, 0.45, idx/(N-1))`) plus a standout accent for high conflict/energy days, clamped to never drop below `LEGIBILITY_FLOOR=0.35`.
- **Contract:** New required `Shell.weight: z.number().min(0).max(1)` on `ShellSchema`, type inferred via `z.infer` (no hand-written interface, CLAUDE.md §1 / ENG-01). Present on every shell including genesis.
- **Paint:** `drawShell` multiplies `shell.weight` into per-glow `energyAlpha` and into star size (`sizeWeight = 0.7 + weight*0.3`, with a floor so standout-but-old days stay legible) — the fade is visible end-to-end. paint reads only `shell.weight` (a Scene value), never a raw DayVector (ENG-02).
- **Determinism guard:** Star `elements` arrays are byte-identical to the pre-change baseline (asserted via committed snapshot deep-equality) — only `radius` and the new `weight` differ, proving zero new `rng()` calls / no RNG-order drift (Pitfall 1).

## Decisions Made
- **Formula:** clamped-minimum-gap hybrid (RESEARCH Pattern 1A, the recommended candidate) over a pure exponential or pure linear falloff — exponential preserves the receding-onion perspective feel vs the mock, the linear term + hard floor guarantees deep rings never collapse.
- **Constants are `[ASSUMED]` and tunable:** `RADIUS_EXP_BASE=0.92`, `RADIUS_EXP_WEIGHT=0.45`, `RADIUS_LIN_WEIGHT=0.55`, `MIN_GAP=0.9/N`, `LEGIBILITY_FLOOR=0.35`. These were accepted as-is at the visual checkpoint (the developer judged ~30 rings read well); no tuning requested.
- **Geometry-only change:** star `elements` arrays deliberately left byte-identical so determinism (same seed + genomeVersion → byte-identical Scene) holds and no rng() drift is introduced.

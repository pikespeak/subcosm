<objective>
Rework the shell depth geometry (VIS-DEPTH) so earlier/frozen day-shells read distinctly all the way toward the core — eliminating the central blob caused by `radius = Math.pow(0.85, idx)` (`src/engine/synthesis.ts:146`) — and emit a per-shell `weight` scalar so older shells fade with age while standout days keep accents (no shell below a legibility floor). This is the synthesis-side, contract-changing half of the phase; it re-baselines the golden snapshot once.

Purpose: Per D-01 (perspective falloff with a guaranteed minimum gap) and D-02 (fade with age, keep highlights), every ring stays individually distinguishable when scrubbing frontier→day-1, judged side-by-side vs `docs/subcosm-universe-mock.html`. The change lives in synthesis because "which days stand out / how deep a ring sits" is data about the day on the `Scene` contract — paint only maps the scalar to pixels (Architectural Responsibility Map, RESEARCH).

Output: New pure helpers `shellRadius(idx, N, prevRadius)` and `shellWeight(idx, N, day)` in `synthesis.ts`; a new `Shell.weight` field on `ShellSchema`; a re-baselined + extended `synthesis.test.ts`; and `paint.ts` `drawShell` multiplying `weight` into per-glow alpha/size so the fade is visible end-to-end.
</objective>

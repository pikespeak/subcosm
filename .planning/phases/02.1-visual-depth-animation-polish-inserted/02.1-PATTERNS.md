### `src/engine/synthesis.ts` (engine, transform) — VIS-DEPTH (D-01, D-02)

**Analog:** the `synthesize()` body itself. The D-01 radius change replaces `Math.pow(0.85, idx)` at **l.146**; the optional D-02 `weight` scalar is emitted in the same fixed-key Shell literal at **l.144-157**.

**Imports pattern** (l.22-31) — engine imports ONLY contracts + rng, never style/paint/Devvit:
```ts

```

**The exact target line** (l.142-157) — the Shell fixed-key literal; `radius` is the D-01 change-point, and a `weight` field (if D-02 picks placement 2a) goes here as another always-present key:
```ts
const shell: Shell = {
  day: day.day,
  radius: Math.pow(0.85, idx),   // ◀── D-01: replace with clamped-min-gap falloff
  meta: { date: day.date, era: day.dominantTheme, theme: day.dominantTheme,
          posts: day.posts, comments: day.comments,
          contributors: day.contributors, conflict: day.conflict },
  elements,
};
```

**Pure-helper precedent to mirror** (l.35-41) — add the D-01 falloff / D-02 weight math as pure `idx`/`N`/metric functions next to these, NOT inside the rng-consuming loop:
```ts
function lerp(a: number, b: number, t: number): number { return a + (b - a) * t; }
function clamp01(v: number): number { return v < 0 ? 0 : v > 1 ? 1 : v; }
```

**RNG-order discipline (CRITICAL, Pitfall 1)** — the per-day rng closure is seeded at **l.89** (`const rng = mulberry32(day.seed)`) and every `rng()` call in the element loop (l.114-125) advances ONE shared stream. **D-01/D-02 must add ZERO `rng()` calls** — compute from `idx`/`N`/already-derived `day.conflict`/`day.energy`. Adding rng() in the loop shifts every downstream element → full re-baseline instead of a `radius`(+`weight`)-only diff.

**Fixed-key literal rule (SYN-02, Pitfall 3)** — see the Element literal at l.128-138 and Shell at l.144-157: every key always present, no conditional keys, for byte-identical key order. A new `weight` key must be present on EVERY shell (genesis included), never `weight?:` conditionally spread.

### `src/engine/contracts/Scene.ts` (contract, schema) — D-02/D-03 OPTIONAL field add

**Analog:** `ShellSchema` (l.45-51) and the per-field-bounded `CoreNodeSchema` (l.12-17). Any new `Shell` field follows the same `z.number().min(0).max(1)` + `z.infer` pattern — **no hand-written interface** (CLAUDE.md §1, ENG-01).

**Bounded-scalar precedent** (l.12-17) — copy this shape for `Shell.weight` / `Shell.energy` / `Shell.momentum`:
```ts
export const CoreNodeSchema = z.object({
  radius: z.number(),
  energy: z.number(),
  hue: z.number().min(0).max(1), // 0..1 hint, not a color
});
export type CoreNode = z.infer<typeof CoreNodeSchema>;
```

**Where the field lands** (l.45-51) — add inside `ShellSchema` as a bounded scalar, then the inferred `Shell` type updates automatically:
```ts
export const ShellSchema = z.object({
  day: z.number().int().positive(),
  radius: z.number(),
  // weight: z.number().min(0).max(1),   // ◀── D-02 (2a) if synthesis emits weight
  // energy: z.number().min(0).max(1),   // ◀── Pitfall 4: per-shell energy scalar
  // momentum: z.number().min(-1).max(1),// ◀── D-03 OPTIONAL drift (extra re-baseline)
  meta: ShellMetaSchema,
  elements: z.array(ElementSchema),
});
export type Shell = z.infer<typeof ShellSchema>;
```

**Contract-add cost:** any field here forces a `synthesis.test.ts` snapshot re-baseline (user-approved for radius). `conflict` is already on `meta` (l.41) — D-03's primary mapping needs NO contract change; only `momentum`/clean per-shell `energy` would.

### `src/client/cosmos/CosmosScene.ts` (client, rAF loop) — VIS-ANIM (D-03, D-04)

**Analog:** `update()` (l.149-160). The harsh `pulse`/`twinkle` constants at **l.157-158** are the exact replacement target.

**Reduced-motion gate to mirror EXACTLY** (l.152-154) — all new shimmer/pulse math MUST stay below this guard (Pitfall 3, PNT-04):
```ts
this.controller?.update();                       // navigation runs even reduced-motion
if (!this.animate || !this.paintResult) return;  // ◀── ALL new pulse math below this line
```

**The target lines** (l.155-159) — replace with shimmer-base + bounded conflict/energy modulation reading `shells[0].meta.conflict` (already on Scene), never a raw DayVector (ENG-02):
```ts
const speed = this.style.motion.speed;
const pulse = 0.55 + 0.45 * Math.sin(time * 0.0022 * speed);  // ◀── D-04: replace (strobes [0.10,1.0])
const twinkle = 0.72 + 0.28 * Math.sin(time * 0.004 * speed); // ◀── D-04: raise base, cap swing
this.renderFrontier(time, pulse, twinkle);
```

**Static-frame parity** (l.140 in `layout()`, l.129 in `repaintFrontier()`) — the reduced-motion/initial frame calls `renderFrontier` with rest values. The new shimmer's REST value (high base, zero swing) must match what these static calls pass, so the non-strobe frame and the animated baseline agree:
```ts
this.renderFrontier(0, this.animate ? 0.8 : 1, this.animate ? 1 : 1);
```

**Frontier-energy caching (Pitfall 4)** — `Shell` has `meta.conflict` but no per-shell energy. Either add `Shell.energy` (contract) or cache the mean element energy ONCE on `PaintResult` (paint.ts `PaintResult`, l.48-57) — do NOT `reduce()` over elements every frame in the hot `update()` path.

**60fps rule** — only `shells[0]` (frontier) is touched per frame; `renderFrontier` (l.167-178) redraws one ignite ring + re-alphas frontier stars. Never iterate `shells[1..]` here (they are baked Images).

## Shared Patterns

### Seeded RNG only (engine determinism)
**Source:** `src/engine/rng.ts` (`mulberry32`, l.24-32)
**Apply to:** `src/engine/synthesis.ts` only — and D-01/D-02 must add ZERO new `rng()` calls.
`Math.random()` is ESLint-banned inside `src/engine/` (ENG-03). Paint variance (D-05) must be a pure function of Scene metrics — no RNG in paint at all.

### Contract split: synthesis Phaser-free, paint never reads raw data
**Source:** header comments `synthesis.ts:1-22`, `paint.ts:10-18`, `CosmosScene.ts:9-11`
**Apply to:** every file in this phase.
- Geometry (`radius`, `weight`) decisions live in `synthesis.ts` (the Scene contract).
- Motion/look (pulse, baked signature) lives in paint, reading `shell.meta.conflict` / a Shell energy scalar — **never `day.conflict`**.

### Zod single source of truth (any new field)
**Source:** `src/engine/contracts/Scene.ts` `CoreNodeSchema` (l.12-17)
**Apply to:** `Shell.weight`/`energy`/`momentum` if added — `z.number().min().max()` + `z.infer`, no hand-written interface. `.parse()` stays at boundaries; never inside synthesis/paint/rAF loop (QA-03 perf).

### Reduced-motion gate (PNT-04)
**Source:** `src/client/cosmos/CosmosScene.ts` `update()` guard (l.154) + `reduced-motion.ts`
**Apply to:** all new VIS-ANIM math — keep it below `if (!this.animate ...) return;`. Static frame uses rest values (`layout()` l.140).

### Bake-to-DynamicTexture (zero per-frame cost)
**Source:** `src/client/cosmos/bake.ts` `bakeShell` (l.29-51) — `dt.draw(objects)`
**Apply to:** D-05 frozen-shell signature. Variance rides each glow object's own alpha/tint/size (Pitfall 2: `dt.draw` alpha/tint args ignored on game objects).

### Reused glow / facet primitives (don't hand-roll)
**Source:** `src/client/cosmos/primitives.ts` `addGlow` (l.55-70), `addFacetStar` (l.90-104)
**Apply to:** D-05 variance — modulate the `radius`/`color`/`alpha` args passed to these existing calls; do not build new draw primitives or per-frame gradients (PNT-03 trap).

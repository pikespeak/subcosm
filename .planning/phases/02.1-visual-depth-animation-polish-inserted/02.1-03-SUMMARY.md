# Phase 02.1 Plan 03: Star Density Floor (VIS-DENSITY) Summary

Raised the per-day star floor so every day-shell reads as a visibly populated cluster — even the quietest simulated days — while preserving the busy/drama/AMA-vs-quiet contrast and full determinism. Requirements addressed: **VIS-DENSITY**.

## What changed

### 1. starCount mapping (`src/engine/synthesis.ts`)

Before:
```ts
function starCount(posts: number, density: number): number {
  return Math.max(5, Math.min(112, Math.round(posts * density)));
}
```
After (two named, documented `[ASSUMED]` constants added near the other tunables):
```ts
export const STAR_FLOOR = 18;
export const STAR_BASELINE = 10;
function starCount(posts: number, density: number): number {
  return Math.max(
    STAR_FLOOR,
    Math.min(112, Math.round(STAR_BASELINE + posts * density)),
  );
}
```

The function stays **PURE** (a function of `posts` and `density` only), monotonic non-decreasing in `posts`, with the **112 cap unchanged**. No `Math.random()`, no Devvit — the `src/engine/` purity rule holds.

### Chosen constants — verified numerically across the beats arc (calm density 0.24)

| beats day | posts (mean) | old stars | new stars |
|-----------|--------------|-----------|-----------|
| day-30 (quiet, "the frontier") | 9 | 5 | **18** |
| day-29 (quiet) | 11 | 5 | 18 |
| day-25 (quiet) | 24 | 6 | 18 |
| day-8 (growth) | 49 | 12 | 22 |
| day-20 (AMA) | 64 | 15 | 25 |
| day-12 (drama) | 92 | 22 | **32** |
| day-19 (growth) | 84 | 20 | 30 |
| fixture day-44 (dense) | 410 | 98→cap 112 | **108** |

Quietest day now 18 stars (target ~≥16–22), drama day clearly denser at 32, densest day 108 < 112 cap.

### 2. Re-baseline (`tests/synthesis-elements-baseline.json`)

This change legitimately alters element COUNTS, so the frozen elements guard (captured for the VIS-DEPTH geometry change, which asserted elements were byte-identical) now fails by design. The baseline was **regenerated** from current `synthesize` output (a throwaway vitest writer test, then removed). Per-shell element counts moved `[5, 5, 0]` → `[108, 60, 0]` (genesis stays core-only). The fixture keeps its **determinism intent**: same seed reproduces the same elements byte-for-byte. The stale "elements unchanged vs VIS-DEPTH" comment in the test was updated to say it is now the determinism re-baseline (a stray non-density rng() call, or any per-element drift at a fixed count, still fails loudly).

### 3. New / updated tests (`src/engine/synthesis.test.ts`)

A new `synthesize density (VIS-DENSITY)` describe block, driven through the **real `generateDayVectors`** with a fixed master seed (`0xc05c01c`) so assertions run over actual emitted DayVectors (jitter included), not raw beat means:

- **min-stars per shell across the full sim arc** — every non-genesis shell has `≥ STAR_FLOOR` stars; genesis (day-1) asserted core-only/exempt.
- **busy > quiet contrast** — drama day-12 has clearly more stars than quiet day-30 (quiet sits exactly at the floor; busy is ≥ ~1.4× denser).
- **cap intact** — no shell exceeds 112.

The Pitfall-1 elements test was renamed/recommented from "byte-unchanged vs the pre-D-01 baseline" to "byte-reproducible vs the re-baselined snapshot (determinism)".

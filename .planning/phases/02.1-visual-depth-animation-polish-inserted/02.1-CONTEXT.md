<decisions>
## Implementation Decisions

### Depth geometry (VIS-DEPTH)
- **D-01: Perspective falloff with a guaranteed minimum gap.** The radius mapping still shrinks toward the center (recent days larger, depth feels real), but every ring keeps a guaranteed minimum radial gap from its neighbor — no ring is ever crushed into the central blob. Rejected: pure-linear equal-spacing (loses the depth/infinity feel, crowds the core at 30+ rings) and a fixed N-day horizon (sacrifices deep-past detail). The new mapping replaces `Math.pow(0.85, idx)` in `src/engine/synthesis.ts`.

### Old-shell visual weight (VIS-DEPTH)
- **D-02: Fade with age, but keep highlights.** Base brightness decreases with shell age for depth atmosphere, but standout days (high conflict / high energy) retain accents so notable moments stay readable when scrubbed to. A minimum legibility floor is guaranteed — no shell fades to illegible. Rejected: uniform fade (loses notable days) and equal-brightness-for-all (kills depth mood).

### Metric → motion mapping (VIS-ANIM)
- **D-03: `conflict` → pulse hardness/amplitude, `energy` → pulse tempo/speed** (`momentum` may add a slow drift). Per-metric mapping, not one combined intensity scalar. The exact curve constants are left to research/planning, but the mapping direction is locked.
- **D-04: Baseline is a gentle shimmer, not a hard on/off blink.** The current `0.55 + 0.45*sin` swing reads as harsh "blinking" — the new baseline is a soft shimmer with data-driven modulation layered on top, never strobing. Still frontier-only per frame (60fps rule intact).

### Frozen-shell character (VIS-ANIM × VIS-DEPTH)
- **D-05: Bake a static per-day metric signature into each frozen shell.** A high-conflict frozen day looks visibly different from a calm one even though it does not animate per frame — the signature is baked into the RenderTexture (zero per-frame cost) and reinforces the per-shell brightness/size tuning from D-02. Frozen shells gain *character*, not *motion*.

### Claude's Discretion
- Exact radius curve (the falloff function + minimum-gap constant) — D-01 fixes the behavior, planner picks the formula.
- Exact metric→parameter curves/constants for the ignite modulation — D-03 fixes the mapping direction only.

</decisions>

## Existing Code Insights

### Reusable Assets
- **Bake-to-RenderTexture path (PNT-03)** already exists for frozen shells → D-05's baked per-day signature rides on it with zero per-frame cost.
- **`drawIgniteRing()` (`paint.ts`)** is the existing ignite-pulse seam → VIS-ANIM modulates the `pulse` argument fed into it from `CosmosScene.update()`.
- **Seeded RNG (`src/engine/rng.ts`)** for any deterministic per-shell variance — no `Math.random()`.

### Established Patterns
- **Synthesis is Phaser-free; paint never touches raw data.** VIS-DEPTH geometry change happens in `synthesis.ts` (the `radius` contract value); VIS-ANIM motion change happens in client paint (`CosmosScene.update()`) reading `shell.energy` + `shell.readout.conflict`.
- **`conflict` + `energy` are already on the Scene shell** → D-03's primary mapping needs no contract change. `momentum`-driven drift would require adding a field to `ShellSchema` → contract change + snapshot re-baseline.

### Integration Points
- **VIS-DEPTH edits the engine contract `radius`** → golden snapshots re-baselined + determinism tests updated (**user approved re-baseline**). Determinism guarantee itself is unchanged: same `seed + genomeVersion` still reproduces identical output; only the baseline shifts.
- **VIS-ANIM edits `CosmosScene.update()`** (frontier-only) and the bake step for frozen-shell character — must keep the 60fps rule and `prefers-reduced-motion` static path intact.

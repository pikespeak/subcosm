<objective>
Make real Reddit activity accumulate into per-community Redis daily counters (DEV-02) and define the pure conflict-composite normalization (DEV-03) from comment-rate + reply-depth proxies. This is the vertical slice that turns the captured trigger payloads (plan 03-01) into stored signal: posts/comments → counters, unique contributors → SET, top threads → ZSET, replies → a reply-depth proxy, and a manipulation-resistant `conflict` function ready for the tick to read.

Purpose: DEV-02 requires triggers to increment Redis daily counters with unique-contributor SET + top-thread ZSET, and DEV-03 requires a conflict composite from comment-rate / reply-depth proxies (no vote stream exists — D-02). The conflict curve is the only genuinely-new design work and must be pure + unit-tested so a few deep replies from one user can't spike it (D-02 intent, security threat T-03-04).
Output: A pure `conflictComposite` function, a `counters` Redis-accumulation service, a single `frontierDay(sub)` helper (the one source of the current/frontier day index, reused by the sweeper in 03-04), and trigger handlers wired to accumulate (replacing the plan 03-01 log-only stubs).
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client webview → own server | same-origin `/api/organism` only; no client-supplied sub id is trusted |
| server response → client UI | the client must not throw on a malformed/hostile payload |
| stored ring → render | rings are parsed at the server read boundary (03-03); the client trusts the inferred type, never re-parsing on the hot path |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-02 | Spoofing / Info disclosure | sub id on /api/organism | mitigate | Derive `sub` from `context.subredditId` server-side; a client cannot request another community's rings (V4) |
| T-03-12 | Tampering | malformed server/network payload → client crash | mitigate | UI `OrganismResponseSchema.safeParse` (never `.parse`/throw); a bad payload routes to the error overlay, never a broken canvas (CLAUDE.md section 6) |
| T-03-13 | Information disclosure | /api/organism exposing more than render needs | mitigate | The OrganismResponse envelope is exactly rings (~25 scalars + seed) + genome/style ids — no secrets, no PII, no images (DEV-05) |
| T-03-08 | Tampering | partial stored ring rendered raw | mitigate | `readAllRings` is the single Redis-read boundary parse (03-03); the client never re-parses in the rAF loop (Pitfall 6) |
| T-03-SC | Tampering | npm/pip/cargo installs | mitigate | No new packages (RESEARCH Package Legitimacy Audit clean — fetch is native, Phaser/Zod pre-existing); slopcheck N/A |
</threat_model>

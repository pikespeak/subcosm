# Phase 3 — UI Design Contract

> Visual and interaction contract for the **Devvit-specific surfaces** of Phase 3 (Devvit Scaffold + Data Layer). Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Scope & Locked Foundation

**This phase does NOT re-design the universe render.** The genesis core, concentric shells, nebula, frontier ignite, the Techno visual system, depth scrubber and HUD readout were designed and shipped in Phases 2 + 2.1. Their visual language is **locked** and lives canonically in `src/client/cosmos-dev/cosmos-dev.css` + `src/client/cosmos/paint.ts` (already passed a prior UI-SPEC). Phase 3 **reuses** that palette / type / motion language verbatim.

Phase 3 adds only **four Devvit-specific UI surfaces** around the locked render:

| # | Surface | What it is | DEV ref |
|---|---------|-----------|---------|
| S1 | **Splash first-frame** | The inline `splash.html` card shown in the Reddit feed before the interactive `game.html` webview loads | DEV-01 |
| S2 | **Interactive post webroot** | How the cosmos render sits inside the (small, embedded, mobile) Reddit post viewport | DEV-01 |
| S3 | **Load / cold-start / error states** | Loading-while-fetching, day-1 genesis-core cold-start (D-04), and offline/fetch-error fallback | DEV-05, D-04 |
| S4 | **Mod install settings form** | Devvit-native settings (genome preset + style + IANA timezone) configured at install | DEV-06, D-03, D-04 |

**Honored hard rules (spec §39–42):** legibility mandatory in every style · respect `prefers-reduced-motion` · mobile-first (Reddit post viewport, 60fps, only frontier animates) · one style per community (set at install). **i18n (CLAUDE.md §7):** every user-facing string and every settings validation error is referenced by an i18n key, not hardcoded language text.

## Spacing Scale

8-point scale (multiples of 4), matching the locked cosmos chrome:

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Glyph→label gaps |
| sm | 8px | Compact element spacing, button-grid gaps |
| md | 16px | Default padding (panels, splash card inset, HUD inset) |
| lg | 24px | Section breaks between control / form groups |
| xl | 32px | Splash vertical rhythm (logo → title → subtitle blocks) |
| 2xl | 48px | Major splash spacing |
| 3xl | 64px | Reserved |

**Exceptions:** **44px** minimum interactive touch target (slider rows, buttons, the splash Start button, select fields) — mandated by the locked chrome and Reddit mobile viewport. Not a spacing-scale value; it is a hit-area floor.

## Color

60 / 30 / 10 split, identical to the locked cosmos chrome (`cosmos-dev.css` `:root`):

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#05040b` page / `#04030a` stage | Near-black space background, splash card ground, all surfaces |
| Secondary (30%) | `#0c0a16`→`#0a0812` gradient, `#1c1636`→`#15102a` (controls/cards) | Panels, splash card body, button/field substrate, overlays |
| Accent (10%) | `--hot #ff8a3d`, `--gold #ffd08a` | See reserved-for list below |
| Destructive | `--conflict #ff5a6e` | Conflict-meter glyphs ONLY (a *data* color, not a UI-error color) |

**Text / hairline tokens (not part of the 60/30/10 — they are ink, not accent):** `--ink #eceaf5` (primary text), `--muted #8b86a8` (keys, sub-labels), `--hair rgba(150,130,220,0.14)` (borders).

**Accent reserved for (explicit, never "all interactive elements"):**
- `--hot #ff8a3d` — the live **FRONTIER** badge, the frontier ignite edge, and the warm (frontier) end of the depth-scrub gradient track.
- `--gold #ffd08a` — the **theme** data field, the daily **goal** line, the **BIG BANG / genesis** badge, and the splash primary **Start** affordance emphasis.

**Critical rule:** ordinary chrome (buttons, fields, the install form) spends NO accent — it uses the secondary substrate + `--hair`. Accent is reserved for frontier/genesis/theme/goal/CTA only. The error state must NOT introduce a new alarm-red; it reuses muted ink (errors are not destructive actions — see Copywriting).

## Per-Surface Visual & Interaction Contract

### S1 — Splash first-frame (`splash.html`)
- **Layout:** vertically centered card on the `#05040b` ground; logo/genesis glyph → title → one-line subtitle → primary **Start** button → muted footer. Mobile-first single column; max content width ~480px, centered on wide.
- **Visual:** background should evoke the cosmos (a faint genesis-core glow behind the card is acceptable, static), but must stay legible (≥4.5:1 text). Replace the boilerplate "Edit src/client/splash.ts" copy + the Docs/Devvit/Discord footer.
- **Motion:** static or a single subtle core pulse; **must honor `prefers-reduced-motion`** (static when reduced). No strobe.
- **Touch:** Start button ≥44px tall, full-width on mobile.

### S2 — Interactive post webroot (`game.html` / cosmos mount)
- **Layout:** the Phaser stage fills the post viewport (`flex:1; min-height:0; touch-action:none`) exactly as `#cosmos-stage`. HUD readout pinned top-left (16px inset), canvas hint bottom-right — both `pointer-events:none`, `z-index:2`, reusing `.hud` / `.canvas-hint`.
- **Aspect / safe area:** assume a small embedded viewport; the render must center on the genesis core and remain legible at the post's default (short) height before any scrub. Respect iOS safe-area insets on the chrome overlays.
- **WebGL-in-iframe risk:** `[ASSUMED]` Phaser/WebGL runs in the Devvit post iframe on mobile — flagged as a pre-work spike (STATE.md). If WebGL is unavailable, the cosmos render falls back to the Canvas2D painter path; the contract here is unaffected (same Scene seam).
- **Touch:** `touch-action:none` so pinch/drag drive the camera (CAM-02), never the page.
- **Note:** the full control panel (nudges/regenerate/seed) is a Phase-4/dev surface — the read-only post in Phase 3 shows the render + HUD only (fetch-on-load, D-01).

### S3 — Load / cold-start / error states
All three render over the stage; all copy via i18n keys.

- **Loading (fetching Ring records):** translucent-dark centered status over the stage reusing `.hud` panel styling (`rgba(5,4,12,0.78)` + blur). A single static "forming" line or a non-strobe core shimmer (reduced-motion → static). Never a spinner that implies failure.
- **Cold-start / day-1 (D-04):** **NOT an empty/error state.** Render the glowing genesis core alone (first star appears on first activity). HUD shows the designed cold-start line (`.hud-empty`, ink-colored) — reuse the existing `T-02-12` cold-start treatment. Must read intentional and beautiful.
- **Error / offline (fetch failed):** translucent-dark `.hud`-style panel, **muted ink only — no alarm-red** (the conflict-red token is reserved for data). State the problem + the next step ("can't reach the cosmos · pull to retry"). Offer a retry affordance (secondary substrate button, no accent).

### S4 — Mod install settings form (Devvit-native, `devvit.json` `settings`)
Host-rendered Devvit settings surface (Developer Portal / install). **Do NOT custom-CSS it** — declare fields with Devvit settings primitives. All `scope: installation` (one config per community — "one style per community"). Field vocabulary confirmed against `@devvit/web` 0.13.4 / Devvit settings (`type`, `name`, `label`, `helpText`, `defaultValue`, `onValidate`).

| Field | type | name | Default | Notes |
|-------|------|------|---------|-------|
| Genome preset | `select` | `genome` | `calm` | Options: Calm / Chaotic / Crystalline. Drives behavior (data, not code). |
| Style | `select` | `style` | `techno` | Options from registered StyleTemplate ids (currently Techno; Crystalline is a Techno variant). One style per community. |
| Community timezone | `string` (IANA) or `select` | `timezone` | (mod's subreddit TZ or `UTC`) | Used by the hourly UTC sweeper to fire the local-midnight tick (D-03). |

- **Validation (i18n keys, CLAUDE.md §7):** `timezone` must be a valid IANA zone → `onValidate` returns i18n key `error.settings.timezone.invalidIana` on failure. `genome` / `style` must be a known preset id → `error.settings.genome.unknown` / `error.settings.style.unknown`. Parse all three at the server boundary with Zod (`SettingsSchema.parse`) before use.
- **Copy:** field labels + helpText reference i18n keys (e.g. `settings.genome.label`, `settings.genome.help`). Each option label explains its character in one phrase (helpText).
- **Defaults are intentional:** Calm + Techno + the mod's local TZ give an immediately beautiful cold-start with zero further input.

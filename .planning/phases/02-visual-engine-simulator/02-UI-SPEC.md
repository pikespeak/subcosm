# Phase 2 — UI Design Contract

> Visual and interaction contract for the **dev-page chrome** that surrounds the cosmos canvas. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Scope boundary (read first):** This contract governs the **chrome only** — the HUD readout panel, the control harness (scrubber + zoom + nudges + regenerate + seed + preset selector), the responsive layout, and the chrome's own motion/contrast. It does **NOT** govern the cosmos artwork (genesis core, shells, nebula, stars, frontier) — that is rendered by the Phaser PAINT layer and is fully data-driven by `StyleTemplate` (CONTEXT.md D-04 mock-parity-elevated, D-05 Crystalline). The cosmos palette is referenced only so the chrome harmonizes with it. The authoritative visual source for the chrome is `docs/subcosm-universe-mock.html` (its `<style>` block + readout/panel markup).

---

## Design System

| Property | Value |
|----------|-------|
| Tool | **none** — vanilla Vite + Phaser + hand-written CSS. No shadcn, no Tailwind, no component library (confirmed: no `components.json`, no `tailwind.config.*`; styling is plain `.css`, e.g. `src/client/game.css`). shadcn gate **not applicable** — the dev page is a single full-bleed canvas with bespoke chrome, not a form/CRUD React surface; shadcn's component model offers nothing for this surface. |
| Preset | not applicable |
| Component library | none — chrome is plain HTML elements styled with CSS, wired to the `render()` API |
| Icon library | **none** — glyphs only. Nudge controls use Unicode glyphs exactly as the mock: `❋` (branch/scatter), `✷` (symmetry/arms), `◐` (hue). Conflict meter uses block glyphs `▮▯`. No icon font / SVG icon set. |
| Font | **Space Grotesk** (chrome UI: labels, buttons, titles) + **Space Mono** (readout values, seed field, timers, captions, axis end-labels). Loaded once via Google Fonts (`wght@400;500;600;700` for Grotesk, `wght@400;700` for Mono), same as the mock. `system-ui, sans-serif` fallback. |

**Self-authored note (anti-AI-slop, D-04):** the mock's chrome is the authored reference. Reuse its exact treatments (translucent readout, faceted nudge buttons, gradient scrub track). Do not substitute generic Material/Bootstrap chrome.

## Spacing Scale

Declared values (must be multiples of 4). The mock uses tight, sub-grid paddings (10–14px) appropriate to a phone-width post viewport; this contract **normalizes them onto a 4px grid** for consistency without changing the visual density.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Glyph→label gap inside nudge buttons; conflict-bar internal; pip→text gap in live feed |
| sm | 8px | Scrub-row gap; nudge-button grid gap; readout line-height rhythm; readout inner padding |
| md | 16px | Default: panel horizontal padding, readout-to-canvas inset, control group internal padding |
| lg | 24px | Section breaks: between scrub-row, nudge group, regenerate/seed group, preset selector |
| xl | 32px | Layout gap between the canvas stage and the control panel on desktop (two-column) |
| 2xl | 48px | Outer page padding / max-width gutter on desktop |
| 3xl | 64px | Reserved — page-level spacing on wide desktop; unused on mobile |

**Exceptions (touch-target floor — non-negotiable, mobile-first):**
- All interactive controls (nudge buttons, regenerate button, preset selector, seed field) have a **minimum 44×44px hit area** even when the visible chrome is smaller. The scrub-slider thumb is visually 16px but its row carries a **44px vertical touch zone**.
- The scrub track itself is **3px** tall (visual hairline) — exception to the grid, matches mock; its hit area is padded to 44px.

## Typography

Mobile-first sizes. Two families, capped weight set. Sizes normalized to whole px from the mock's fractional values.

| Role | Size | Weight | Line Height | Family | Usage |
|------|------|--------|-------------|--------|-------|
| Body | 16px | 400 (regular) | 1.5 | Space Grotesk | Control labels ("Shape today's frontier:"), preset selector text, general chrome copy. Bumped from the mock's 10–11px (post-viewport) to a legible 16px floor for the **dev page** per WCAG; the eventual Reddit-post embed may re-tighten, but the dev harness must be comfortably readable. |
| Label | 14px | 600 (semibold) | 1.4 | Space Grotesk | Nudge-button labels, regenerate button, section headers (e.g. "Shape today's frontier:"), HUD field keys. |
| Mono / data | 14px | 400 (regular) | 1.55 | Space Mono | **All HUD readout values** (date / era / theme / stars / comments / contributors), seed input field, axis end-labels (TODAY / BIG BANG), nudge sub-captions, caption line, conflict bar. |
| Display | 20px | 600 (semibold) | 1.2 | Space Grotesk | HUD primary line ("DAY 47" + FRONTIER/BIG-BANG badge) and any page title. |

**Two weights only:** 400 (regular) and 600 (semibold). The mock's 500/700 are collapsed to this pair (700 → 600 for the HUD day-line and theme; 500 unused). No other weights.

**Line heights:** body 1.5, heading/display 1.2, mono data 1.55 (matches mock's readout rhythm; mono benefits from looser leading for scannability over the busy canvas).

## Color

The chrome lives **over a dark, glowing, animated canvas** — legibility against that background is the central challenge and a **hard rule** (CONTEXT.md D-02, subcosm-spec "legibility mandatory"). Chrome surfaces are therefore **translucent dark panels with backdrop-blur**, not transparent overlays. Values lifted from the mock's authored palette.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#04030a` / page `#05040b` (near-black space) | The canvas/stage background and page background. The chrome reads *on top of* this. |
| Secondary (30%) | `rgba(5,4,12,0.6)` over the canvas; solid panel gradient `linear-gradient(180deg,#0c0a16,#0a0812)` for the control panel | HUD readout surface (translucent + `backdrop-filter: blur(3px)`); the control-panel substrate; nudge-button fill `linear-gradient(180deg,#1c1636,#15102a)`. Hairline borders `rgba(150,130,220,0.14)`. |
| Text — primary | `#eceaf5` (`--ink`) | HUD day-line, value emphasis, button labels. |
| Text — muted | `#8b86a8` (`--muted`) | HUD field keys, end-labels, captions, sub-text. |
| Accent (10%) | warm-gold `#ffd08a` + ignite-orange `#ff8a3d` (`--hot`) | **Reserved list below.** |
| Destructive / alarm | conflict-red `#ff5a6e` | The conflict meter bar glyphs **only** (`▮` filled cells when conflict is high). Not a button color — no destructive *actions* exist in this phase (see Copywriting). |

**Accent reserved for (explicit — never "all interactive elements"):**
1. The **theme** value in the HUD (`#ffd08a`) — the one highlighted data field.
2. The **FRONTIER / BIG-BANG badge** chip background (`#ff8a3d` for FRONTIER, `#ffd08a` for BIG-BANG).
3. The **frontier "solidifies in HH:MM:SS" countdown** text + its meter fill (`#ffd08a` text, orange gradient fill) — signals the live/active day.
4. The **goal line** for the frontier day (e.g. "Goal: tame conflict below 0.3") — rendered in accent gold so the day's stakes are unmissable (GAME-01 legibility).
5. The **scrub-slider track gradient** end-stop (`#fff7e6` warm-white → core) — the only place the gradient is decorative; deep end is indigo `#6366f1`.

Everything else (buttons, borders, body text, preset selector) uses dominant/secondary/text-muted — **accent is never spent on ordinary chrome.**

**Contrast floor (hard rule):** all HUD text and control labels must hit **≥ 4.5:1** against the *worst-case* bright patch of canvas behind them (e.g. the genesis core or an AMA cluster). This is why HUD/panel surfaces are opaque-enough translucent fills with blur, not pure-transparent text — the panel fill guarantees the ratio regardless of what the canvas is doing behind it. The conflict-red `#ff5a6e` is used for glyph *bars*, not small body text, so its lower contrast is acceptable.

Dev-harness copy. Note the mock is German (it previews the eventual r/Subcosm post); **per CLAUDE.md all artifacts/code are English** — the dev page chrome is authored in **English**. (A later i18n pass governs the shipped post; the dev page itself is English.)

| Element | Copy |
|---------|------|
| Primary CTA | **"Regenerate"** (with secondary line "new seed") — the dev page's main action; re-dices the simulator within the scripted beats (SIM-01) and re-renders. |
| Section header — nudges | **"Shape today's frontier"** |
| Nudge buttons | **"Scatter"** (`❋`, sub: "+ wild") · **"Arms"** (`✷`, sub: "+ structure") · **"Hue"** (`◐`, sub: "shift") |
| Genome preset selector label | **"Genome"** — options: **Calm · Chaotic · Crystalline** |
| Seed field | label **"Seed"**, placeholder shows the current numeric seed; "Regenerate" reads this field (re-entering a seed reproduces the identical universe — Success Criterion 3) |
| Axis end-labels | left **"TODAY"** · right **"BIG BANG"** (mock: HEUTE / URKNALL) |
| Frontier goal line (HUD, frontier day only) | **"Goal: {goal text}"** e.g. "Goal: tame conflict below 0.3" — gold accent, only shown when the focused shell is the frontier (GAME-01). |
| HUD badges | **"FRONTIER"** (frontier day) · **"BIG BANG"** (genesis/day-1) |
| HUD field keys | date · era · theme · stars · comments · people · conflict |
| Empty state heading | **"Day 1 — the first post"** |
| Empty state body | The cold-start day-1 (one ring, maybe one star) is **a designed state, not an error** (Success Criterion 4 + Phase 5 SC-4). Copy: *"A universe begins. One post, one spark — scrub outward as the community grows."* The genesis core must read as intentional and beautiful, never as "no data". |
| Error state — bad seed | If the seed field holds a non-parseable value: *"That seed isn't valid — using a random one instead."* (non-blocking; the page never shows a broken canvas — it falls back to a random valid seed and regenerates). |
| Error state — render/sim failure | If `DayVectorSchema.parse()` rejects simulator output (should never happen in-phase): inline dev banner *"Simulator produced invalid data — check the console."* — dev-only, since this surface is the dev harness. |
| Reduced-motion note | No copy change; behavior only (see Motion). |

**Destructive actions:** **none in this phase.** "Regenerate" overwrites the on-screen universe but persists nothing (no stored state in Phase 2 — Redis arrives in Phase 3), so it needs **no confirmation dialog**. Nudges are additive and reversible-by-regenerate. There is nothing to delete, no irreversible action, no confirmation pattern required.

## Motion & Reduced-Motion (chrome only)

> Cosmos-canvas motion (frontier ignite, twinkle, rotation) is the PAINT layer's concern, governed by `StyleTemplate.motion` (`frontierOnly`, `speed`) — out of scope here. This section covers **chrome transitions only.**

| Element | Motion | Reduced-motion (`prefers-reduced-motion: reduce`) |
|---------|--------|------|
| Nudge button press | `transform: translateY(1px) scale(.985)` over 80ms; border-color over 150ms | Keep press feedback (it's a discrete tap response, not ambient/strobe) — allowed. |
| Live-feed line entry | `rise` 350ms ease (fade + 3px up) | **Disabled** — lines appear instantly, fully opaque (mock already does `animation:none;opacity:1` under the media query). |
| Meter fill width | 350ms ease transition | Width snaps instantly; no animated fill. |
| Scrub → readout update | instant (no transition on text) | unchanged. |
| Countdown timer | per-second text update | unchanged (a clock, not animation). |

**Hard rule (PNT-04):** under reduced-motion the entire surface — chrome **and** canvas — presents a **static, non-strobe frame**. No ambient pulsing, no auto-advancing live feed, no twinkle. The chrome honors this via the rules above; the canvas honors it via `StyleTemplate.motion`.

## Layout (responsive, mobile-first)

The dev page previews the eventual Reddit-post viewport; **mobile-first is the primary target** (CONTEXT.md). The full-bleed cosmos canvas dominates; chrome floats over it (HUD) or stacks beneath it (controls).

**Mobile (≤ 640px) — single column, the canonical layout:**
- Full-width **canvas stage** at top (mock: ~362px tall in a 392px phone frame), `touch-action: none` so pinch-zoom drives the camera not the page.
- **HUD readout** absolutely positioned **top-left over the canvas**, `max-width: 66%`, translucent + blur, `pointer-events: none` (never intercepts canvas gestures). **Always visible** (D-02 hard rule) — never collapsed, never behind a toggle.
- A bottom-right **canvas hint** ("◀ scrub outward = back in time"), mono, muted, `pointer-events:none`.
- **Control panel** stacked below the canvas, in this vertical order with `lg`(24px) gaps:
  1. Scrub row: `TODAY ——slider—— BIG BANG`
  2. "Shape today's frontier" + 3-up nudge button grid (`grid-template-columns: repeat(3,1fr)`, `sm` gap)
  3. Frontier countdown meter row
  4. Regenerate button + Seed field (inline row)
  5. Genome preset selector (Calm / Chaotic / Crystalline)
  6. Live feed (last 3 nudge lines)

**Desktop (> 640px) — two-column:**
- Canvas stage grows to fill the left/main area (square-ish, capped by viewport height); HUD stays pinned top-left **over** the canvas.
- Control panel moves to a **right rail** (fixed ~320–360px), same vertical control order, separated from the canvas by `xl`(32px). Outer page gutter `2xl`(48px).
- The HUD remains an over-canvas overlay in **both** layouts — it never migrates into the rail (legibility/parity with the post embed).

**Interaction affordances (CAM-01/02, QA-01):**
- **Scrub slider** = time (drag flies through shells); **pinch / scroll-wheel** = zoom; **click/tap a shell** = focus+zoom that shell. These three coexist (D-01). The slider and the click-focus must stay in sync (clicking a shell moves the slider; dragging the slider changes focus).
- All controls reachable one-handed on mobile (thumb zone); the panel never requires horizontal scroll.

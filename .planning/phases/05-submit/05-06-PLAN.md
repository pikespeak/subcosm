<objective>
Make the goal->steer->reveal loop EXPERIENCEABLE in a single judging session. Today the payoff (frontier freezes -> scored -> reward glyph) only happens at the overnight tick and is mod-gated, so a judge sees a beautiful but passive viewer and never feels the game. This plan adds two client-only, no-new-gameplay pieces:

1. **Live goal-progress meter (SUB-04 legibility / GAME-03):** the HUD shows how the live frontier is tracking against the day's goal (current scored `measured` vs `threshold`, direction, achieved-yet?), and it visibly moves when the player nudges — so steering reads as causal (which it now is).
2. **In-session reveal preview (SUB-02 demo hook):** a button ANY viewer can press that plays a LOCAL freeze -> score -> reward-glyph sequence on the current frontier (no server call, does not tick the real community), clearly labelled as a preview. This collapses the slow overnight loop into a ~30s payoff a judge can trigger themselves.

Output: a `revealPreview.ts` module (pure scoring/sequence logic + meter model) + HUD/game wiring + the preview button + reduced-motion-safe animation, reusing the existing engine score() and the 04-04 reward-glyph paint.
</objective>

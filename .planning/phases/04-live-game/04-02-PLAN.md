remaining budget; a nudge UI that re-synthesizes the acting user's frontier immediately and a HUD that

contribution→outcome link — the realtime broadcast to OTHER viewers lands in plan 03.

near-real-time), GAME-03 (steering measurably moves the day toward/away from the goal, legibly), and

application of steerGain at the tick.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → POST /api/steer | Untrusted nudge body (`param`, `amount`); ids must come from server context, not the body |
| client UI ← /api/steer + /api/organism responses | App-originated but `safeParse`d at the client boundary (never throw) |
| concurrent users → steer hash | Many users mutate the same per-day hash; must aggregate atomically, no lost updates |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-04 | Spoofing / Elevation | POST /steer identity | mitigate | Derive `sub = context.subredditId`, `userId = context.userId` server-side (V4); the only client input is `param`+`amount`. A client cannot nudge as another user or write another sub |
| T-04-05 | Tampering | ActionBudget bypass (replay/concurrent nudges) | mitigate | Atomic `incrBy`-then-compare in recordNudge (Pattern 4); over-cap calls return remaining=0 and do NOT aggregate — a user cannot exceed the cap via concurrency (TOCTOU closed) |
| T-04-06 | Tampering | Oversized/garbage steer `amount` skews the shared frontier | mitigate | `SteerRequestSchema` clamps `amount` to [-1,1] and `param` to the enum; `.parse()` at the route boundary (V5) rejects a hostile bias before it reaches the steer hash |
| T-04-07 | Tampering | Steer-hash clobbering (one user overwrites another) | mitigate | `hIncrBy` SUMS each contribution (never `hSet`); concurrent users accumulate, no lost updates (D-04 no-clobber) |
| T-04-08 | DoS | Nudge flooding past the cap | mitigate | Budget gate caps per-user nudges/day (default 3); the steer hash + budget keys are deleted/expired at the tick |
| T-04-SC | Tampering | npm/pip/cargo installs | mitigate | No new packages (RESEARCH Package Legitimacy Audit: zero new deps); no install task |
</threat_model>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| fixtures → synthesize | In Phase 1 the only data crossing into the engine is test fixtures, validated by `DayVectorSchema.parse` in `tests/fixtures.ts`. synthesize itself trusts the z.infer types (no .parse inside). |
| rng → synthesis | `mulberry32(day.seed)` is the sole entropy path; any other randomness would break determinism. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering (determinism leak) | synthesis.ts entropy | mitigate | Seed from `day.seed` (NOT index `i*9973`); mulberry32 is the only RNG; ESLint Math.random ban (Plan 01) enforces it; byte-identical test (toEqual + JSON.stringify) proves it. |
| T-02-02 | Tampering (key-order non-determinism) | Element/Shell construction | mitigate | Each Element/Shell built as ONE fixed-key object literal (all keys always present) — no conditional spreads; JSON.stringify equality test catches key-order drift (Pitfall 3). |
| T-02-03 | Tampering (style coupling breaks Scene seam) | synthesis.ts output | mitigate | `Element.hue` is a 0..1 hint, never a color; synthesis imports no style/paint code (ENG-02); grep + ESLint cross-dir import ban enforce it. A color leak here is unfixable without a contract change. |
| T-02-04 | Information disclosure (empty-array crash) | topThreads reduction | mitigate | `Math.max(0, ...day.topThreads)` guards the empty array → nbig 0 (Pitfall 2); noUncheckedIndexedAccess (inherited) forces the guard. |
| T-02-05 | Spoofing (RNG misuse) | rng.ts | accept | mulberry32 documented non-cryptographic; used only for visual determinism; never seeds secrets/tokens. Low risk — no security context in a pure engine. |
</threat_model>
